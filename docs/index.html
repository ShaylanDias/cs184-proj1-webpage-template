Section 1: Rasterization

Part 1: Rasterizing single-color triangles

In order to rasterize a triangle I first calculate its bounding box by getting the minimum and maximum x and y values. I then iterate through this bounding box and run a check on each point to see if it is within the triangle. I do this by using the line test on each point to see if the point is above or below the line by taking the dot product of a vector perpendicular to the line and the vector from the first point of the line to the point being checked. If the point is either “above” or “below” all three of the lines then it is in the triangle and I call fill_pixel to fill its color.

This is no slower than checking each sample in the bounding box of the triangle since it does exactly that (calculates the bounding box and checks only the samples within it).

Part 2: Antialiasing triangles

In order to supersample when rasterizing I took uniformly distributed samples within each pixel equal to the sampling number. To accomodate these extra samples I multiplied the size of the sample buffer by the number of samples per pixel. This created space to store all of the samples for each pixel without creating any extra data structures. This new sample buffer could then be indexed using the sample number in addition to the x and y coordinates of the pixel.

To then render the rasterized triangle I averaged the colors of the samples at each pixel location and then pushed those to the frame buffer. The super sampling is useful because it takes into account that the color value at the center of a pixel may not reflect all of the information that needs to be present to render the image (especially in the case of angled edges and skinny points of triangles). The averaged values create a smoother blend for edges to avoid aliasing (jaggies!) and make it less likely that a pixel that should have color will be missing which causes the gaps in the purple and red triangles seen above.

Section II: Sampling

Part 4: Barycentric Coordinates

Barycentric coordinates are a coordinate positioning system within a triangle. Each point in the system consists of three weights, each of which is relative to one of the three points of the triangle. The coordinate system then positions the point using each weight as a proportion of how close it is to each corner (with all of the weights adding up to 1 if the point is within the triangle). We can see this at work in the image below where each point is colored using each weight for the color assigned to each corner. The colors are very distinct close to the corner where they are weighted close to 1, but blend smoothly toward the middle.

Part 5: "Pixel sampling" for texture mapping

Pixel sampling means converting a point on the screen to a point within a separate texture which may be different from the screen itself and then interpolating a value from that point within the texture to display on the screen. I implemented it by solving the linear system for each x/y point and its conversion to the u/v space of the texture then sampling from that point using either nearest or bilinear sampling. For nearest neighbor this involves simply casting the u/v point to an integer and sampling at that point. For bilinear it involves looking at the four nearest integer points and returning a weighted average of them based on how near they are to the point.

In the above examples the bilinear sampling clearly creates more continuous looking lines than the nearest sampling. There is a large difference between the methods when drawing skinny lines where a nearest sample could miss the line entirely in a spot where it should be somewhat present (especially when sampling down from a larger texture). The bilinear pixel interpolation helps to alleviate this by ensuring some part of small elements near textures are represented.

Part 6: "Level sampling" with mipmaps for texture mapping

Level sampling is the idea that we can measure how much a texture is stretched in order to be rendered onto an image by measuring the delta between the u/v space points after they are calculated from two adjacent x/y points. Using this stretch we can sample from different textures which are at different resolutions for performance reasons since it is unnecessary to use a high-resolution texture on a small image when the details will not carry over.

Adding to the number of samples per pixels will help with antialiasing but at a large performance tradeoff since doing 16 samples instead of 1 is essentially increasing the number of computations necessary by 16x. However, it does a strong job antialiasing. Bilinear pixel sampling is a seemingly cheaper way of doing antialiasing though it may not be as strong alone but it will increase speed. Using level sampling we will increase our memory usage but only marginally since the mipmaps loaded will be larger than just the standalone texture.
